在本地创建分支并关联到远程分支 #

创建本地分支，并切换到本地分支上，同时将本地分支和远程分支进行关联。

git checkout -b 本地分支 origin/远程分支

 

放弃所有的文件修改 #

git checkout 的作用是放弃掉还没有加入到暂存区的修改。此命令不会删除新创建的文件，因为新创建的文件还没有被跟踪（tracked）,因此需要git clean,删除当前目录下没有被track过的文件和文件夹

git clean -df
git checkout .

 

git remote add #

git add 和git commit 相当于将本地的文件都上传到了本地仓库，但是还没有提交到远端仓库；

这时执行git remote add，就是将本地仓库与远端仓库建立一个链接,下面两种方式都可以

git remote add origin git@github.com:yourName/repositoryname.git

git remote add origin https://github.com/yourName/repositoryname.git
  • 为远端仓库所起的名字，一般都是叫origin

 

本地关联远程分支 #

  • 使用git在本地新建一个分支后，需要做远程分支关联。如果没有关联，git会提示你添加关联。
  • 关联目的是在执行git pull, git push操作时就不需要指定对应的远程分支，否则，git pull的时候，就会提示你添加关联。

例：

git branch --set-upstream-to=origin/fixBug fixBug

 

强制用远程分支更新本地 #

正确的做法如下：

git fetch --all

git reset --hard origin/master // 远程分支名称

git commit时提示用户名错误 #

  1. 修改作者的默认配置
git config --global user.name  wutengteng1
git config --global user.email  wutengteng@jd.com
  1. 更新完成后，查看配置信息确保准确无误

git config --list

  1. 配置信息正确后需要修改commit信息，

1 ）使用git log命令查看git 提交记录，找到之前的版本号

2 ）git reset --soft 到之前的版本重新commit，push

git reset --soft e99da8c5c9d59de34448e375b836f1bd3c7acf73

更换了git密码后，git push时错误 #

解决：在钥匙串中把git相关的钥匙串删除，重新执行git命令，提示输入账号密码，输入正确的账号密码就可以了
